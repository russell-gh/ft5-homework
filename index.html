Positions and their uses

position: static; - the default/canceller
  • All elements are 'statically positioned' by default. This means that they use the box model (the 'logs in the river') system
  • This declaration can therefore be used to cancel any of the others using css specificity to override: If you want an absolutely positioned element to stop flying around you can set that element to position static using css
top, bottom, left & right: < some css length/unit >
  • These have no effect if the element is statically positioned!
  • When something is positioned in a 'non-static' way then its positional coordinates are where the element's box would normally be
  • To adjust this we use the CSS properties top, bottom, left & right

  • These measurements are 'relative' to either the 'last non-statically-positioned-ancestor' OR, if you don't have one of those, the <body> tag.

    This does mean that if you have drop down menus on a sticky header then you have a position absolute menu inside a position sticky header, and all measurements for the menu will be relative to the header (if that is the last non-statically positioned ancestor)!

  • Mentally, I think of The Weakest Link when I do position. Every time I put a non-static position on something I think 'BANK!' and now I know my positions are going to be relative to this element.
position: relative; - for visual micro-adjustments
  • This effects not where element boxes are but where they appear to be (so not on the draw/layout phase but on the paint phase.
  • This is often used for 'micro adjustments', when lining things up visually (e.g. the logo with the links in a nav
  • If all you're doing is moving an element's apparent position then transform: translate() is a more efficient option
  • It is often used as the 'last non-statically-positioned-ancestor' because it comes with few complications. If you're wanting to 'fly' a position absolute element around an area then give that area position: relative;
position: fixed; - sticking things to the viewport, like 'social media tabs'
  • Fixed takes an element onto a new composite layer (e.g. 'out of the flow of the page' - the helicopter with the grabby claw)
  • It positions the element relative to the viewport
  • This was how we used to do sticky headers but because it comes out of the flow of the page it presents the problem of content going up underneath it and having to use padding to push it back out again (all very well until the content is dynamic!)
  • It's still useful for things like side-menus (think 'social media tabs on side of viewport')
position: sticky; - sticky headers/footers
  • Sticky is an upgrade to fixed
  • Like relative, sticky leaves the element box where it is and just 'visually moves' the element during the paint phase.
  • Sticky can also be used on any element, not just the edges of the viewport (see here opens in a new tab)
  • To get sticky to work you need to state a top or bottom value for it, reative to the lst non-statically positioned ancestor or, failing that, the body tag.
position: absolute; - putting an element over others (e.g. modal)
  • Absolute is for putting an element over the top of another. (think 'modals', 'content that slides over images', etc.
  • Absolute comes with consequences and is difficult to control: Use it only for the use cases mentioned above.
